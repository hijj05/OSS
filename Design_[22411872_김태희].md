# 1. Instroduction

과거부터 사람들은 서로 필요한 물건을 교환하거나 정보를 공유하며 공동체를 형성해 왔다. 특히 같은 지역에 거주하는 사람들끼리는 생활에 필요한 물품을 나누고 취미나 관심사가 같은 사람들과 모임을 가지며 지역 사회를 발전시켜 왔다. 그러나 현대 사회에서는 이웃 간의 교류가 점차 줄어들고 있으며 사용하지 않는 물건이 방치되거나 지역 주민들 간의 소통 기회가 부족해지는 문제가 발생하고 있다.

최근 중고거래 플랫폼과 온라인 커뮤니티의 활성화로 개인 간 거래와 모임 활동이 증가하고 있지만 대부분의 서비스는 중고거래 또는 모임 기능 중 하나에만 집중되어 있다. 또한 지역 인증 절차가 복잡하거나 신뢰할 수 없는 거래 상대방으로 인한 사기 문제, 모임 정보 관리의 어려움 등 다양한 문제점이 존재한다. 이러한 문제를 해결하고 지역 주민들이 더욱 쉽고 안전하게 교류할 수 있는 환경을 제공하기 위해 동네 기반 중고거래 및 모임 플랫폼인 “WeShare”를 개발하게 되었다.

WeShare가 추구하는 첫 번째 목표는 안전하고 편리한 지역 기반 중고거래 환경을 제공하는 것이다. 사용자는 자신의 동네를 설정한 후 가까운 지역 주민들과 거래할 수 있으며 검색 기능을 통해 쉽고 빠르게 거래를 진행할 수 있다. 이를 통해 거래 거리를 최소화하고 사용자 간 신뢰성을 높일 수 있다.

두 번째 목표는 지역 주민 간의 활발한 소통과 커뮤니티 형성이다. 사용자는 운동, 독서, 게임, 스터디 등 다양한 주제의 모임을 개설하거나 참여할 수 있다. 이를 통해 같은 관심사를 가진 사람들을 연결하고 지역 사회의 유대감을 강화할 수 있다.

세 번째 목표는 사용자 친화적인 통합 플랫폼 구축이다. 중고거래와 모임 서비스를 하나의 시스템 안에서 제공함으로써 사용자는 여러 애플리케이션을 사용할 필요 없이 필요한 기능을 한 곳에서 이용할 수 있다. 또한 직관적인 인터페이스와 체계적인 데이터 관리를 통해 누구나 쉽게 사용할 수 있는 서비스를 제공하고자 한다.

본 시스템의 주요 타겟은 대학생, 직장인, 지역 주민 등 모두를 타겟으로 하며 특히 중고물품 거래를 자주 이용하거나 새로운 사람들과 취미 활동 및 정보 교류를 원하는 사용자들에게 유용한 서비스를 제공할 수 있을 것으로 기대된다.

아래는 WeShare 시스템 개발의 세 번째 단계인 Design에 관한 내용으로써 실제 시스템 구현에 직접적으로 관여하는 모든 요소들의 구조와 기능을 구체적으로 설계하는 과정을 다루고 있다. 본 문서의 모든 세부 사항은 실제 구현 과정에서 사용될 데이터 구조, 화면 구성, 기능 설계 및 소스 코드와 일관성을 유지하는 것을 목표로 한다.

## 1. UserProfileActivity
### Description

UserProfileActivity 클래스는 사용자의 프로필 정보와 활동 내역, 평점 등을 관리하는 클래스이다.

### Attributes

totalScore : Float
: 사용자가 받은 총 평점을 저장한다.

reviewCount : int
: 사용자가 받은 리뷰의 개수를 저장한다.

score : float
: 평점을 저장한다.

comment : String
: 사용자 리뷰 내용을 저장한다.

average : float
: 평균 평점을 저장한다.

### Methods

addReviewToScreen() : void
: 사용자가 받은 리뷰를 화면에 출력한다.

## 2. RegisterActivity
### Description

RegisterActivity 클래스는 회원가입 기능을 담당하는 클래스이다.

### Attributes

newUserId : String
: 회원가입 시 입력한 아이디를 저장한다.

newPassword : String
: 회원가입 시 입력한 비밀번호를 저장한다.

nickname : String
: 사용자가 입력한 닉네임을 저장한다.

### Methods

registerUser() : void
: 새로운 사용자 정보를 등록한다.

duplicateCheck() : boolean
: 아이디 중복 여부를 확인한다.

validateInput() : boolean
: 입력값의 유효성을 검사한다.

## 3. LoginActivity
### Description

LoginActivity 클래스는 사용자 인증 및 로그인 기능을 담당하는 클래스이다.

### Attributes

inputId : String
: 사용자가 입력한 아이디를 저장한다.

inputPassword : String
: 사용자가 입력한 비밀번호를 저장한다.

loginStatus : boolean
: 현재 로그인 상태를 저장한다.

### Methods

checkUser() : boolean
: 입력된 사용자 정보를 데이터베이스와 비교하여 확인한다.

validatePassword() : boolean
: 입력된 비밀번호가 올바른지 검증한다.

## 4. DataBase
### Description

DataBase 클래스는 사용자, 게시글 및 기타 데이터를 저장하고 관리하는 클래스이다.

### Attributes

userDB
: 사용자 정보를 저장하는 데이터베이스이다.

postDB
: 게시글 정보를 저장하는 데이터베이스이다.

### Methods

getString() : void
: 데이터를 조회한다.

putString() : void
: 데이터를 저장한다.

updateData() : void
: 기존 데이터를 수정한다.

deleteData() : void
: 데이터를 삭제한다.

## 5. CategoryActivity
### Description

CategoryActivity 클래스는 게시글 카테고리를 관리하고 분류하는 클래스이다.

### Attributes

categoryName : String
: 카테고리 이름을 저장한다.

posts : String
: 해당 카테고리에 포함된 게시글 정보를 저장한다.

### Methods

loadPostsByCategory() : void
: 선택된 카테고리에 해당하는 게시글을 불러온다.

addPostBox() : void

## 6. PostDetailActivity
### Description

PostDetailActivity 클래스는 게시글의 상세 내용을 조회하고 거래 요청 기능을 제공하는 클래스이다.

### Attributes

title : String

price : String

content : String

: 작성자 이름을 저장한다.

### Methods

tradeRequest() : void
: 해당 게시글에 대한 거래 요청을 수행한다.

## 7. WriteActivity
### Description

WriteActivity 클래스는 사용자가 새로운 게시글을 작성하는 기능을 담당하는 클래스이다.

### Attributes

title : String
: 게시글 제목을 저장한다.

price : String
: 상품 가격을 저장한다.

content : String
: 게시글 내용을 저장한다.

### Description

MainActivity 클래스는 애플리케이션의 메인 화면을 담당하며 게시글 목록과 추천 정보를 관리하는 클래스이다.

### Description

# 3. Sequence diagram
## login
<img width="496" height="567" alt="Image" src="https://github.com/user-attachments/assets/dba72dc3-d4fe-45ec-8685-af034d2ba61b" />

애플리케이션에 로그인하는 과정을 나타낸다. 사용자는 로그인 화면에서 아이디와 비밀번호를 입력한 후 로그인 버튼을 선택한다. LoginActivity는 입력된 정보를 DataBase에 전달하고 입력 정보와 저장된 회원 정보를 비교하여 인증 결과를 반환한다. 인증에 성공하면 사용자는 Main View 화면으로 이동하고 인증에 실패할 경우 오류 메시지를 출력하고 로그인 화면에 머무르게 된다.

## Registration
<img width="630" height="597" alt="Image" src="https://github.com/user-attachments/assets/70053373-88cf-4080-8044-b1034da3a622" />

신규 사용자가 계정을 생성하는 과정을 나타낸다. 사용자는 회원가입 화면에서 아이디, 비밀번호 등의 정보를 입력한다. RegisterActivity는 입력값의 유효성을 검사하고 데이터베이스에 중복된 아이디가 존재하는지 확인한다. 중복 검사가 완료되면 새로운 사용자 정보를 데이터베이스에 저장한다. 회원가입이 성공하면 로그인 화면으로 이동하며, 실패할 경우 오류 메시지를 표시한다.

## Post
<img width="652" height="550" alt="Image" src="https://github.com/user-attachments/assets/9b43efe4-2dc8-4d7c-9e56-dc368a45400c" />

사용자가 새로운 게시글을 등록하는 과정을 나타낸다. 사용자는 게시글 작성 화면에서 제목, 가격, 내용 및 카테고리를 입력한 후 등록 버튼을 선택한다. WriteActivity는 입력된 데이터를 검증한 후 DataBase에 저장 요청을 보낸다. 데이터베이스에 게시글이 정상적으로 저장되면 성공 메시지를 반환하며 게시물이 등록된다.

## Trade
<img width="545" height="553" alt="Image" src="https://github.com/user-attachments/assets/5a99e6e7-bdf2-4d32-972a-389cc6ff0b9a" />
사용자가 특정 게시글에 대해 거래를 요청하는 과정을 나타낸다. 사용자는 게시글 상세 화면에서 거래 버튼을 선택한다. 거래 요청 정보가 상대방 사용자에게 전달되며 거래가 진행된다.

## Profile
<img width="642" height="422" alt="Image" src="https://github.com/user-attachments/assets/53a8ec48-89ff-4f9a-ab71-70bea749208a" />

사용자가 상대방 또는 자신의 프로필을 확인하는 과정을 나타낸다. 프로필 화면을 요청하면 데이터베이스에서 정보를 반환하여 사용자에게 보여준다.


## Category
<img width="596" height="401" alt="Image" src="https://github.com/user-attachments/assets/68fc1f4e-2edc-4af3-8952-a4bf58e50414" />

사용자가 카테고리별로 게시물을 찾는 과정을 나타낸다. 데이터베이스에서 카테고리 목록을 받아 띄우고 사용자가 선택한 카테고리의 게시물들을 보여준다

## Review
<img width="467" height="353" alt="Image" src="https://github.com/user-attachments/assets/5f21b5a2-4f83-4b9d-92ac-2e8d45bdd284" />

사용자가 상대방을 평가하는 과정을 나타낸다. 사용자는 상대의 프로필 화면에서 평점과 리뷰 내용을 입력한다. UserProfileActivity는 입력된 정보를 데이터베이스에 저장하도록 요청한다. 데이터베이스는 리뷰 정보를 저장하고 사용자의 평균 평점을 갱신한다. 이후 갱신된 평점과 리뷰 내역이 프로필 화면에 반영된다.

# 4. State machine diagram
<img width="618" height="577" alt="Image" src="https://github.com/user-attachments/assets/cb509d40-841a-4d08-bc70-a90a08d0f8f7" />

WeShare의 흐름과 화면 전환 과정을 시각적으로 나타낸 것이다. 사용자는 애플리케이션 실행 시 로그인(Login) 화면에 진입하며, 회원이 아닌 경우 회원가입(Registration) 화면으로 이동하여 계정을 생성할 수 있다. 회원가입이 정상적으로 완료되면 다시 로그인 화면으로 돌아오며, 로그인에 실패할 경우 로그인 화면에 머무르게 된다.

로그인에 성공하면 메인 화면(Main View)으로 이동한다. 메인 화면은 애플리케이션의 중심 화면으로 사용자는 게시글 조회, 검색, 게시글 등록, 프로필 조회, 위치 설정 등의 기능을 이용할 수 있다.

검색 버튼을 선택하면 검색(Search) 상태로 이동하며, 사용자가 입력한 키워드에 따라 게시글(Post)을 조회할 수 있다. 또한 게시글 작성 버튼을 선택하면 게시글 업로드(Upload Post) 상태로 이동하여 새로운 게시글을 등록할 수 있다. 

메인 화면에서 특정 게시글을 선택하면 게시글 상세 조회(View Post) 화면으로 이동한다. 사용자는 게시글의 상세 정보를 확인할 수 있으며, 거래 버튼을 클릭하여 거래(Trade)를 진행할 수 있다. 거래가 완료된 이후에는 리뷰(Review)를 작성하여 거래 상대방에 대한 평가를 남길 수 있다.

사용자는 위치 버튼을 선택하여 위치(Location) 설정 화면으로 이동할 수 있으며 프로필(Profile) 화면에서는 사용자 정보, 평점, 리뷰 내역 등을 확인할 수 있으며 로그아웃 버튼을 통해 로그아웃(Logout) 상태로 이동하여 애플리케이션 이용을 종료할 수 있다.
